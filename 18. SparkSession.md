
# SparkSession

## Overview

- `SparkSession` is the entry point for reading data, working with metadata, and configuring Spark applications
- It unifies `SQLContext`, `HiveContext`, and `SparkContext` into a single API.

## Creating a SparkSession

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("MyApp") \
    .master("local[*]") \
    .config("spark.some.config.option", "value") \
    .getOrCreate()
```

## Key Builder Parameters

### `.appName(name)`
- Sets the application name shown in Spark UI
- Helps identify jobs in cluster monitoring

### `.master(url)`
- Specifies cluster manager connection
- **Options**:
  - `local` - Single thread
  - `local[*]` - All available cores
  - `local[4]` - 4 cores
  - `spark://host:port` - Standalone cluster
  - `yarn` - YARN cluster
  - `mesos://host:port` - Mesos cluster

### `.config(key, value)`
- Sets Spark configuration properties
- Can be chained multiple times
- **Example**: `.config("spark.executor.memory", "4g")`

### `.enableHiveSupport()`
- Enables Hive support for reading Hive tables
- Requires Hive dependencies

### `.getOrCreate()`
- Returns existing SparkSession or creates new one
- Recommended over `create()` for reusability

## Common Configuration Options

| Parameter | Description | Default | Example |
|-----------|-------------|---------|---------|
| `spark.app.name` | Application name | None | "MyApp" |
| `spark.master` | Cluster manager URL | local[*] | "yarn" |
| `spark.executor.memory` | Memory per executor | 1g | "4g" |
| `spark.executor.cores` | Cores per executor | 1 | 4 |
| `spark.driver.memory` | Driver memory | 1g | "2g" |
| `spark.sql.shuffle.partitions` | Shuffle partitions | 200 | 100 |
| `spark.default.parallelism` | Default parallelism | cores * 2 | 8 |
| `spark.serializer` | Serialization library | JavaSerializer | KryoSerializer |

## Example: Complete Configuration

```python
spark = SparkSession.builder \
    .appName("AdvancedApp") \
    .master("yarn") \
    .config("spark.executor.memory", "4g") \
    .config("spark.executor.cores", "4") \
    .config("spark.driver.memory", "2g") \
    .config("spark.sql.shuffle.partitions", "100") \
    .config("spark.serializer", "org.apache.spark.serializer.KryoSerializer") \
    .enableHiveSupport() \
    .getOrCreate()
```

## Stopping SparkSession

```python
spark.stop()
```

Always stop SparkSession to release resources properly.
