# Partitioners in Spark

- Partitioners in Spark determine how elements in an RDD or DataFrame are distributed across partitions. 
- It is used while performing `wide transformations` that require `shuffling` data across the cluster.
- Efficient partitioning is crucial for performance, especially for operations like joins and groupBy.
- Use `rdd.<wide_transform>().partitioner.partitionFunc.__name__` to check the partitioner used.
- Here wide_transform() can be any wide transformation like `reduceByKey()`, `groupByKey()`, etc.

## Hash Partitioner

The **HashPartitioner** assigns elements to partitions based on the hash value of the key. It is commonly used for key-value RDDs.

### How it works 
- The partition is determined by `hash(key) % numPartitions`. 
- This ensures that the same key always goes to the same partition.

### Advantages
- Simple and fast; good for evenly distributed keys.

### Limitations
- May cause data skew if keys are not uniformly distributed.
- e.g. if many keys hash to the same value, one partition may become overloaded.


## Range Partitioner

The **RangePartitioner** divides elements into partitions based on sorted ranges of keys.

### How it works 
- Keys are sorted, and ranges are assigned to partitions so that each partition gets a roughly equal number of elements.
- It is useful for ordered data and range queries.
- Range boundaries are determined by `sampling the keys`, and then dividing them into approximately equal-sized groups.
- For this `precheck` a spark job is ran before the actual job to determine the ranges.

### Advantages
- Useful for ordered data and range queries; helps avoid data skew.

### Limitations
- Requires keys to be comparable; may be slower due to sorting.

## Custom Partitioner
- You can define your own partitioning logic by creating a custom partitioner function.
- This function should take a key as input and return a partition index.
- Useful when you have specific requirements for data distribution.

```python
def custom_partitioner(key):
    if key < "banana":
        return 0
    else:
        return 1

data = [("apple", 1), ("banana", 2), ("cherry", 3), ("date", 4)]
rdd = spark.sparkContext.parallelize(data)

# Here 2 is the number of partitions in the output RDD
# And custom_partitioner is a function that defines the partitioning logic
output_rdd = rdd.partitionBy(2, custom_partitioner)

# Show data distribution across partitions
for i in range(output_rdd.getNumPartitions()):
    partition_rdd = output_rdd.mapPartitionsWithIndex(lambda idx, iterator: [list(iterator)] if idx == i else [])
    partition_data = partition_rdd.collect()
    print(f"Partition {i}: {partition_data[0] if partition_data else []}")
```


## Wide Transformations and Default Partitioners

Some common wide transformations in Spark include:

- `groupByKey()`: Uses `HashPartitioner` by default.
- `reduceByKey()`: Uses `HashPartitioner` by default.
- `sortByKey()`: Uses `RangePartitioner` by default.
- `join()`: Uses the partitioner of the parent RDD if available; otherwise, defaults to `HashPartitioner`.
- `cogroup()`: Uses the partitioner of the first parent RDD if available; otherwise, defaults to `HashPartitioner`.

#### HashPartitioner

| cherry  | 6792823539822217417   | 1                    |

*(Note: Actual hash values may differ between runs and Python versions.)*

#### RangePartitioner

With 2 partitions and sorted keys, the **RangePartitioner** first sorts the keys and then divides them into ranges so that each partition contains a contiguous block of keys. For example:

- Partition 0: keys less than or equal to "banana" — ("apple", 1), ("banana", 2)
- Partition 1: keys greater than "banana" — ("cherry", 3), ("date", 4)

The partition boundaries are determined by dividing the sorted list of keys into approximately equal-sized groups. This ensures that each partition receives a range of keys, which is useful for operations that benefit from ordered data.

This ensures keys are grouped by range, which is useful for ordered processing.

## Summary

Choosing the right partitioner improves Spark job efficiency and scalability. Use HashPartitioner for random keys and RangePartitioner for ordered or range-based data.
