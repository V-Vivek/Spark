
# Commonly Used Functions in Spark

## 1. version
Returns the version of Spark being used.

```python
spark.version
# Output: '3.5.0'
```

## 2. sparkContext
Provides access to the SparkContext, which is the entry point for low-level Spark functionality.

```python
sc = spark.sparkContext
print(sc.appName)
```


## 3. conf
- Provides access to Spark configuration settings.
- Used to get or set Sparks runtime configuration properties.

```python
spark.conf.set("spark.sql.shuffle.partitions", "200")
value = spark.conf.get("spark.sql.shuffle.partitions")
print(value)
```

## 4. udf
Creates user-defined functions for custom operations on DataFrames.

#### For Python/DataFrame UDFs:
```python
from pyspark.sql.functions import udf
from pyspark.sql.types import IntegerType

@udf(returnType=IntegerType()) # Define a UDF that returns the IntegerType
def double_salary(x):
    return 2 * x

df = spark.createDataFrame([('Robert', 35000), ('James', 25000)], ('name string, salary int'))
df = df.withColumn('salary', double_salary(df.salary)) # Replaces salary with its double
df.show()

# Output:
+------+------+
|  name|salary|
+------+------+
|Robert| 70000|
| James| 50000|
+------+------+
```

#### For SQL UDFs:
- Use the same `double_salary` function defined above.

```python
# spark.udf.register('udf_name', function)
spark.udf.register("double_salary_sql", double_salary) 
df.createOrReplaceTempView("employees")
spark.sql("SELECT name, double_salary_sql(salary) AS salary FROM employees").show()
```

- Create new UDF for SQL:
```python
from pyspark.sql.types import StringType

# spark.udf.register('udf_name', function, returnType)
spark.udf.register("to_upper_sql", lambda x: x.upper(), StringType())
spark.sql("SELECT to_upper_sql(name) AS name FROM employees").show()
```


## 5. catalog
Provides access to the Catalog API for managing databases, tables, and functions.

#### Database functions:
```python
spark.catalog.listDatabases()
spark.catalog.currentDatabase()
spark.catalog.setCurrentDatabase("database_name")
```

#### Table functions:
```python
spark.catalog.listTables("database_name")                   # List tables in a database
spark.catalog.tableExists("table_name", "database_name")    # Check if table exists
spark.catalog.listColumns("table_name", "database_name")    # List columns in a table
spark.catalog.refreshTable("table_name")                    # Refresh table metadata
spark.catalog.refreshByPath("path/to/table")                # Refresh table by path
spark.catalog.isCached("table_name")                        # Check if table is cached
spark.catalog.cacheTable("table_name")                      # Cache a table
spark.catalog.uncacheTable("table_name")                    # Uncache a table
spark.catalog.clearCache()                                  # Clear all cached tables
spark.catalog.recoverPartitions("table_name")               # Recover partitions of a table
```

#### View functions:
```python
spark.catalog.dropTempView("view_name")                    # Drop a temporary view
spark.catalog.dropGlobalTempView("view_name")              # Drop a global temporary view
```
#### Function functions:
```python
spark.catalog.listFunctions("database_name")               # List functions in a database
spark.catalog.registerFunction("func_name", func)          # Register a new function
```

## 6. newSession
- Creates a new SparkSession with an isolated SQL configuration.
- It has separate SQL configurations, temporary views, and UDFs from the original session.
- But it shares the same underlying SparkContext and table cache.
- Useful for running independent operations without affecting the main session.

```python
new_spark = spark.newSession()
new_spark.conf.set("spark.sql.shuffle.partitions", "100")
```

## 7. stop
Stops the SparkSession and releases resources.

```python
spark.stop()
```
