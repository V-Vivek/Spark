# Introduction

Spark is an open-source, distributed computing system for fast, flexible large-scale data processing with easy cluster programming.

## Spark vs Hadoop

- It is wrong to compare Spark and Hadoop directly.
- Spark replaces MapReduce, not Hadoop as a whole. 
- It still needs storage (like HDFS) and a resource manager (like YARN), 
- But it can also integrate with other storage systems (such as S3) and resource managers (like Mesos or Kubernetes).


## Spark vs MapReduce

### MapReduce 
- Is slow (disk I/O)
- Complex (chained jobs)
- Limited (mainly Java)

### Spark 
- Is faster (in-memory)
- Simpler (concise APIs)
- Supports multiple languages


## Three Key Pillars of Spark

### 1. General Purpose

- Spark supports interactive, batch, streaming, and ML workloads on one platform.
- High-level APIs in Java, Scala, Python, and R.
- Rich libraries for SQL, streaming, ML, and graph processing.
- Integrates with various storage systems (HDFS, S3, Cassandra, etc.).
- Works with different cluster managers (YARN, Mesos, Kubernetes, or standalone).

### 2. In-Memory

- Spark stores intermediate data in RAM for faster, iterative, and interactive processing.
- Reduces disk I/O and speeds up data processing tasks.


### 3. Compute Engine

- Spark efficiently manages distributed resources, scheduling, and fault tolerance for scalable data processing.
- Optimizes execution plans for better performance.
- Handles node failures gracefully to ensure reliable processing.



## Spark Use Cases
- **Data Processing:** ETL, data cleaning, and transformation.
- **Real-Time Analytics:** Monitoring, fraud detection, and alerting.
- **Machine Learning:** Building and deploying ML models at scale.
- **Graph Processing:** Analyzing and processing large-scale graph data.
- **Batch Processing:** Large-scale data processing jobs.
- **Streaming Analytics:** Processing and analyzing real-time data streams.
- **Interactive Data Analysis:** Ad-hoc querying and data exploration.
