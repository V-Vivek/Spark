# Broadcast Variables in Spark

- Broadcast variables are used to efficiently share small, read-only data across all worker nodes in a cluster. 
- Instead of sending the data with every task, Spark distributes the broadcast variable only once to each node.
- This reduces data shuffling and communication overhead.

## Key Points

- **Purpose:** Share small lookup tables or configuration data across tasks.
- **Immutability:** Broadcast variables are read-only.
- **Efficiency:** Data is cached on each node, minimizing network traffic.
- **Usage:** Use `SparkContext.broadcast(value)` to create a broadcast variable.

## Example

```python
# Python example
broadcastVar = spark.sparkContext.broadcast([1, 2, 3])
rdd.map(lambda x: broadcastVar.value[x])
```

## When to Use

- When you need to use the same data in multiple tasks.
- When the data is too large to be sent with every task.

## Limitations

- Broadcast variables cannot be modified by tasks.
- Should be used for read-only data only.
