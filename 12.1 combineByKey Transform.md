# combineByKey
- It lets you aggregate data by key.
- You can define exactly how values for each key are combined.
- Use `combineByKey` for complex aggregations, not just simple reductions.
- It gives you more control than basic aggregation methods.

## Comparison: `combineByKey` vs `reduceByKey`

### How `reduceByKey` Works
- `reduceByKey` merges values for each key using an associative reduce function.
- It combines values within each partition first, then merges results from different partitions.
- The same function is used for both merging within partitions and across partitions.
- Useful for simple aggregations like sum, min, max, etc.
- But in `combineByKey`, you can specify different functions for these steps.

### How `combineByKey` Works

`combineByKey` requires three functions:

1. **createCombiner**: 
    - For each first occurrence of a key, converts a value `V` from the original RDD into the initial value `C`
    - This function is called once per key.

2. **mergeValue**: 
    - For each subsequent occurrence of a key, merges a new value `V` into the previously computed value `C` within a partition.
    - This function is called for each value associated with a key within a partition.

3. **mergeCombiners**: 
    - Merges two `C`s from different partitions.
    - This function is called to combine results from different partitions.
    - The aggregation logic/function of `mergeCombiners` can be different from `mergeValue`, this is a key difference from `reduceByKey`.

- Useful for complex aggregations like computing averages, means, or custom data structures.

### Example Usage

Suppose you have an RDD of (key, value) pairs and want to compute the average for each key:

```python
# Sample RDD with 3 partitions
rdd = sc.parallelize([("a", 1), ("b", 2), ("a", 3), ("b", 4), ("a", 5), ("b", 6), ("a", 7), ("a", 8), ("a", 9)], 3)
# Lets assume the partitions are:
# Partition 1: [("a", 1), ("b", 2), ("a", 3)]
# Partition 2: [("b", 4), ("a", 5), ("b", 6)]
# Partition 3: [("a", 7), ("a", 8), ("a", 9)]

# Using reduceByKey to calculate average
average = rdd.reduceByKey(lambda x, y: (x + y) / 2)
print(average.collect())

# Output from each partition:
# Partition 1: [("a", 2.0), ("b", 2.0)]
# Partition 2: [("b", 5.0), ("a", 5.0)]
# Partition 3: [("a", 8.0)]

# Final output after merging partitions:
# [('a', 5.0), ('b', 3.5)]  # Incorrect average due to how reduceByKey works

# Using combineByKey to calculate average
combined = rdd.combineByKey(
    lambda v: (v, 1),   # createCombiner
    lambda c, v: (c[0] + v, c[1] + 1),  # mergeValue
    lambda c1, c2: (c1[0] + c2[0], c1[1] + c2[1])   # mergeCombiners
)
averages = combined.mapValues(lambda x: x[0] / x[1])
print(averages.collect())
# Output from each partition:
# Partition 1: [("a", (4, 2)), ("b", (2, 1))]
# Partition 2: [("b", (10, 2)), ("a", (5, 1))]
# Partition 3: [("a", (24, 3))]

# Final output after merging partitions:
# [('a', 5.5), ('b', 4.0)]  # Correct average
```

### Summary
Use `reduceByKey` for simple reductions. Use `combineByKey` when you need more control over the aggregation process or when the aggregation logic is more complex.
